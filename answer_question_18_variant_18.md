# Ответ на контрольный вопрос №18 (Вариант 18)

**Студент:** Платонов Иван
**Группа:** 454  
**Полная формулировка вопроса:**  
Как ограничить количество записей для одного клиента в день до трёх (как в варианте 2), используя только SQL-ограничения, без триггеров? Реализуемо ли это?

## Развёрнутый ответ

Ограничить количество записей для одного клиента в день до трёх, используя только чистые SQL-ограничения (без триггеров, хранимых процедур или внешней логики приложения), **невозможно** в классической реляционной модели MySQL 8.0 и более ранних версиях.

### Почему это нереализуемо только через DDL-ограничения?

Ограничения CHECK в MySQL не могут содержать подзапросы и не могут обращаться к другим строкам таблицы или выполнять агрегацию. Конструкция вида:

```sql
CHECK (
    (SELECT COUNT(*) FROM appointments a 
     WHERE a.client_id = client_id AND DATE(a.appointment_datetime) = DATE(CURDATE())) <= 3
)
```

не сработает, так как MySQL запрещает подзапросы в CHECK-ограничениях (они допускаются только в триггерах или хранимых процедурах). Ограничение UNIQUE проверяет уникальность, но не количество. Внешний ключ также не решает задачу подсчёта.
