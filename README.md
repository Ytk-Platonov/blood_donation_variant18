## Описание предметной области

Проектирование базы данных для системы онлайн-записи доноров в благотворительный фонд. Доноры имеют группу крови и резус-фактор. Система хранит дату последней сдачи крови и автоматически проверяет обязательный перерыв не менее 60 дней между донациями. Для доноров, регулярно сдающих кровь, предусмотрены сертификаты. Основное бизнес-правило: запись на новую сдачу возможна только если с момента последней сдачи прошло 60 и более дней.

## Основные сущности базы данных

Таблица donors (доноры): группа крови, резус-фактор, дата последней сдачи, количество донаций.  
Таблица donation_slots (слоты для сдачи): дата и время, максимальное количество доноров.  
Таблица appointments (записи): связь донора со слотом, статус записи.  
Таблица donor_certificates (сертификаты доноров): тип сертификата, дата получения.  
Таблица donation_history (история сдач): фактическая дата сдачи, объём крови, отметка о выполнении.

## Три основных запроса (примеры)

1. Запрос с JOIN трёх таблиц: вывод всех записей с именем донора, группой крови, датой слота и статусом.  
2. Запрос с GROUP BY и HAVING: список доноров, сдавших кровь более 3 раз, с подсчётом их сертификатов.  
3. Запрос с подзапросом: доноры, которым уже разрешена следующая сдача (прошло 60+ дней), но они ещё не записались.

Полные тексты запросов находятся в папке queries.
